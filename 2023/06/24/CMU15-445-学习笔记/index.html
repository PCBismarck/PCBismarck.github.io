hash 的常用算法

static hashing(静态)

LINEAR PROBE HASHING

Single giant table of slots. Resolve collisions by linearly searching for the next free slot in the table.

​ → To determine whether an element is present, hash to a location in the index and scan for it.

​ → Must store the key in the index to know when to stop scanning.

​ → Insertions and deletions are generalizations of lookups.

当发生冲突时，沿着hash到的位置继续向下循环扫描，直到找到第一个空的槽位插入或者遍历完整个表(此时表已经满了，无法插入)

删除时使用墓碑法: 在需要删除的元素位置上设置标记，代表该位置的元素已经失效(逻辑删除)

Note: 线性探针哈希法，与线性哈希(动态hash算法)不同

ROBIN HOOD HASHING

Variant of linear probe hashing that steals slots from “rich” keys and give them to “poor” keys.

​ → Each key tracks the number of positions they are from where its optimal position in the table.

​ → On insert, a key takes the slot of another key if the first key is farther away from its optimal position than the second key.

在hash的槽位上设置一个距离标识，标识其距离最开始hash到的位置的距离，新插入的值如果在寻找新槽位的过程中发现有比自己的距离小的值，则会取代其位置，并将该位置后的所有值向后移动一位(代价非常大，基本不会使用)

CUCKOO HASHING

Use multiple hash tables with different hash function seeds.

​ → On insert, check every table and pick anyone that has a free slot.

​ → If no table has a free slot, evict the element from one of them and then re-hash it find a new location.

Look-ups and deletions are always O(1) because only one location per hash table is checked.

Best open-source implementation is from CMU.

实质上是使用多个hash表，对于一个值，分别对其在不同的hash表上进行hash，找到空位则直接插入；如果没有找到空位则选择其中一个hash表，取代原来表中的元素，然后对被取代的元素重新执行该hash算法(可能会导致全部的元素都需要重新hash)

dynamic hashing(动态)

CHAINED HASHING

Maintain a linked list of buckets for each slot in the hash table.

Resolve collisions by placing all elements with the same hash key into the same bucket.

​ → To determine whether an element is present, hash to its bucket and scan for it.

​ → Insertions and deletions are generalizations of lookups.

最简单的链式hash算法，但其中引入了bucket的概念，bucket size = 1 就是一个链表式的hash

EXTENDIBLE HASHING

Chained-hashing approach where we split buckets instead of letting the linked list grow forever.

Multiple slot locations can point to the same bucket chain.

Reshuffle bucket entries on split and increase the number of bits to examine.

​ → Data movement is localized to just the split chain.

可拓展的hash

大体上就是通过不同的bit位去划分不同bucket负责的hash值域，如果最后一张图中的第一个bucket满了，此时他的hash要检测的范围就会变成 2 bit，即 00xxx…和01xxx…会分别hash到两个不同的bucket上，将原来的一个bucket分散到两个bucket上，增大bucket的容量，而不改变其他的bucket的内容

问题: global bit 和 local bit 分别设置在哪？

LINEAR HASHING

The hash table maintains a pointer that tracks the next bucket to split.

​ → When any bucket overflows, split the bucket at the pointer location.

Use multiple hashes to find the right bucket for a given key.

Can use different overflow criterion:

​ → Space Utilization

​ → Average Length of Overflow Chains

线性hash，动态哈希算法

这个hash算法主要是为了未来可能发生的扩容进行准备，实际上会将hash的值域翻倍，并启用一个新的hash算法，如上图中，红色分割线之前的槽位以后将使用hash2进行hash，之后的槽位仍然使用hash1(在发生扩容的时候需要对原来的槽位重新进行hash，如原来的第一个bucket0是有8和20两个值，重新hash2后20移动到了bucket4)
